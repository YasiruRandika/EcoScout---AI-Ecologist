Specifications
    Step 1 of 8
    1
    📱 User Captures Input User Device
    The mobile PWA simultaneously captures audio from the microphone, JPEG frames from the camera, and GPS coordinates. Echo cancellation is enabled to prevent feedback loops. The user can mute the microphone at any time without dropping the WebSocket connection.
    PCM 16kHz mono JPEG 640x480 @ 1fps GPS coordinates Echo cancellation ON
    2
    🔗 Data Sent via WebSocket User Device
    A single persistent WebSocket connection carries all data to Cloud Run. Audio chunks are sent as binary frames every 100ms. Camera frames and GPS updates are sent as JSON frames. This single-connection architecture avoids reconnection overhead and maintains session state cleanly.
    Binary: PCM ~3200B / 100ms JSON: {type:"image", data:b64} JSON: {type:"gps", lat, lng} /ws/{user_id}/{session_id}
    3
    ☁ Cloud Run Receives and Routes Cloud Run
    FastAPI WebSocket endpoint on Cloud Run receives all incoming frames. The upstream asyncio task decodes and routes each frame to the LiveRequestQueue. Separate asyncio tasks handle video ready and ecology update events concurrently without blocking the main bidirectional stream.
    upstream_task() downstream_task() LiveRequestQueue Session affinity
    4
    ✨ EcoScout Agent Processes via Gemini Live Gemini AI
    The ADK Runner feeds audio, video frames, and GPS into Gemini Live API's bidirectional streaming session. The EcoScout agent maintains a naturalist persona and proactively identifies species without the user having to ask. Native audio output is streamed back in real time.
    gemini-live-2.5-flash-native-audio Bidi WebSocket stream Proactive reasoning Native audio output
    5
    🔧 Agent Calls Tools Cloud Run
    When the agent identifies a species or needs data, it invokes FunctionTools. identify_specimen() uses Gemini 3 Pro Image and Google Search. record_observation() saves to Firestore and queues an ecology event. get_area_species_checklist() queries iNaturalist. get_weather_context() queries Open-Meteo.
    identify_specimen() record_observation() get_area_species_checklist() calculate_biodiversity_metrics()
    6
    📊 Ecology Events Pushed to Client User Device
    The Ecology Notification Task continuously drains the _ecology_events async queue and sends ecology_update WebSocket messages. The client's ecology-viz.js updates the Live Survey Dashboard in real time, rendering D3.js food webs, accumulation curves, Shannon diversity index, and conservation alerts.
    ecology_update WS event D3.js food web Shannon H' index Accumulation curve
    7
    🎥 Async Video Generation via Veo 3.1 Gemini AI
    generate_nature_video() submits a contextual nature documentary prompt to Veo 3.1 as a long-running async operation. The Video Notification Task polls until completion, uploads the MP4 to Cloud Storage, generates a signed URL, and sends a video_ready WebSocket event back to the client.
    veo-3.1-generate-preview Long-running operation video_ready WS event Cloud Storage MP4
    8
    🌿 Client Receives All Outputs End State
    The full expedition experience is delivered: audio narration plays via AudioWorklet, field guide entries with AI-generated illustrations appear in the journal, generated videos play in the media gallery, and the live survey dashboard shows real-time biodiversity metrics - all through a single persistent WebSocket connection.
    Audio playback (AudioWorklet) Field guide illustrations Video in gallery Expedition summary
    10 Agent FunctionTools
    🔬
    identify_specimen()
    Observation
    Identifies a species from the current camera frame using Gemini 3 Pro Image for visual analysis, then verifies the identification against Google Search for accuracy and additional ecological context.
    Gemini 3 Pro Image Google Search iNaturalist
    📝
    record_observation()
    Observation
    Saves a species observation to Cloud Firestore with GPS coordinates, timestamp, confidence, and ecological notes. Simultaneously queues an ecology event to update the live survey dashboard with new species data.
    Cloud Firestore _ecology_events queue Geolocation
    📈
    calculate_biodiversity_metrics()
    Biodiversity
    Computes ecological diversity indices from recorded session observations. Calculates Shannon-Wiener Diversity Index (H'), Simpson's Diversity Index (1-D), species richness, and evenness scores.
    Session observations Shannon H' Simpson 1-D
    🦋
    get_area_species_checklist()
    Biodiversity
    Queries the iNaturalist API to retrieve the expected species checklist for the current geographic area. Returns community-verified observations with taxonomy, conservation status, and seasonal presence for gap analysis.
    iNaturalist REST API GPS coordinates Taxonomy data
    🌤
    get_weather_context()
    Biodiversity
    Retrieves current meteorological data from Open-Meteo for the observation location. Provides temperature, humidity, wind speed, precipitation, and UV index to contextualise species activity patterns.
    Open-Meteo API GPS coordinates Ecological context
    📖
    generate_field_entry()
    Media
    Generates a richly illustrated field guide entry using Gemini 3 Pro Image's interleaved text and image output mode. Produces scientific illustrations alongside ecological notes, habitat description, and conservation status.
    Gemini 3 Pro Image Interleaved output Cloud Storage
    🎞
    generate_nature_video()
    Media
    Submits an async video generation request to Veo 3.1 with a contextual nature documentary prompt based on observed species and habitat. The Video Notification Task polls the long-running operation and delivers an MP4 to Cloud Storage.
    Veo 3.1 Cloud Storage _video_ready_events
    +
    extend_video()
    Media
    Extends a previously generated nature video by an additional 7 seconds using Veo 3.1's video extension capability. Maintains visual and narrative continuity with the original clip for deeper exploration of a captured natural process.
    Veo 3.1 extension +7s continuation Cloud Storage
    🔍
    google_search()
    Data
    ADK built-in Google Search grounding tool. Used to verify species identifications, retrieve scientific literature, check conservation status, and ground all ecological claims with citations. Critical for preventing hallucination in species-sensitive contexts.
    ADK built-in Google Search API Citation grounding
    📋
    create_expedition_summary()
    Data
    Compiles a complete expedition journal from all recorded observations, biodiversity metrics, field guide entries, and generated media for the current session. Produces a structured report saved to Cloud Storage with a shareable signed URL.
    Cloud Firestore Cloud Storage Signed URL
